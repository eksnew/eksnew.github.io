3. Prob. 3203. Find Minimum Diameter After Merging Two Trees - Hard

You are given two 2D integer arrays edges1 and edges2 of lengths n - 1 and m - 1, respectively, where edges1[i] = [ai, bi] indicates that there is an edge between nodes ai and bi in the first tree and edges2[i] = [ui, vi] indicates that there is an edge between nodes ui and vi in the second tree.

You must connect one node from the first tree with another node from the second tree with an edge.

The diameter of a tree is the length of the longest path between any two nodes in the tree.

This calculation stems from a fundamental property of trees:

  • A tree with nodes always has edges because it is a connected graph without cycles.

  • Since the input edges1 contains all the edges of the first tree (and its length is ), the total number of nodes in the tree can be determined by adding 1 to the length of edges1. Similarly, the same logic applies for edges2.

3.1.2.2. Key Concepts

  1. Merging Two Trees with Minimum Diameter:

    • To minimize the resulting tree’s diameter after merging, we carefully choose which nodes to connect between the two trees.
    • Intuitively, connecting “central” nodes of each tree ensures that the resulting tree remains as balanced as possible. These central nodes are often close to the centroids of the respective trees.

  2. Centroid of a Tree:

    • A centroid is a node that, if removed, splits the tree into subtrees, each containing no more than half the original tree’s vertices.
    • Connecting centroids minimizes the maximum distance to any node in the resulting merged tree.

  5. Optimized Approach to Find Tree Diameter:

    • A naive approach to finding the diameter involves iterating over all nodes, calculating the farthest node for each, and keeping track of the longest path. However, this results in complexity and is computationally expensive.
    • The optimized solution uses a two-step Breadth-First Search (BFS):

      • Start BFS again from to find the farthest node . The distance between and is the tree’s diameter.

  6. Why Two BFS Calls Are Sufficient:

    • Case 1: If the starting node lies on the diameter, the BFS will directly find the other endpoint.
    • Case 2: If the starting node is not on the diameter, the farthest node found must lie on the diameter. From there, the second BFS will correctly calculate the diameter’s endpoints.
3.1.2.3. Intuition Behind the Merging Strategy

  • The main insight is balancing the trees by connecting nodes that minimize the heights of their respective subtrees.

  • By considering only half the diameters of the original trees, the merging process ensures that the resulting tree’s longest path is not unnecessarily large.
