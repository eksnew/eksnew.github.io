Sliding Window

Posted on 2024-11-21 Edited on 2024-12-14

1. Definition

Sliding Window problems are problems in which a fixed or variable-size window is moved through a data structure, typically an array or string, to solve problems efficiently based on continuous subsets of elements. This technique is used when we need to find subarrays or substrings according to a given set of conditions.

1.1. How to Identify Sliding Window Problems

  • These problems generally require Finding Maximum/Minimum Subarray, Substrings which satisfy some specific condition.
  • The size of the subarray or substring ‘K’ will be given in some of the problems.
  • These problems can easily be solved in time complexity using nested loops, using sliding window we can solve these in O(n) Time - Complexity.
  • Required Time Complexity: O(N) or O(Nlog(N))
  • Constraints: , If is the size of the Array/String.

2. Prob. 2516: Take K of Each Character From Left and Right

You are given a string s consisting of the characters 'a', 'b', and 'c' and a non-negative integer k. Each minute, you may take either the leftmost character of s, or the rightmost character of s.

Return the minimum number of minutes needed for you to take at least k of each character, or return -1 if it is not possible to take k of each character.

  • Sliding Window Idea:
    • The window [l, r] represents a substring of s that is being kept while all other characters outside the window (to the left of l and right of r) are removed.
    • The goal is to maximize the length of this substring while ensuring that after removing the substring, the counts of 'a', 'b', and 'c' outside the window are all at least k.
  • For every character at the right end of the window (s[r]), decrement its count in count since it is now part of the window.

If the minimum count in count falls below k, it means the substring [l, r] is too long, and we need to shrink it by moving the left pointer l to the right. For each move, we increment the count of s[l] since it is no longer part of the window.

The length of the characters removed is len(s) - (r - l + 1) because the substring [l, r] has length (r - l + 1). Keep track of the minimum characters removed.

3. Prob. 2762. Continuous Subarrays

You are given a 0-indexed integer array nums. A subarray of nums is called continuous if:

Let i, i + 1, ..., j be the indices in the subarray. Then, for each pair of indices i <= i1, i2 <= j, 0 <= |nums[i1] - nums[i2]| <= 2. Return the total number of continuous subarrays.

A subarray is a contiguous non-empty sequence of elements within an array.

The brute force solution involves iterating through all possible subarrays of the input array nums and checking if each subarray is continuous. For each subarray, we calculate the maximum and minimum elements and check if their difference is at most 2. If the condition is satisfied, we increment the count of continuous subarrays.

This solution employs a sliding window technique combined with a frequency map to efficiently calculate the number of valid subarrays where the absolute difference between the maximum and minimum elements is at most 2. Here's a detailed breakdown of the approach:

Using a Frequency Map The frequency map (freq) is used to maintain a count of the elements currently within the sliding window. This allows us to dynamically calculate the maximum and minimum elements in the window:

  • The maximum is obtained by taking the largest key in the map (max(freq)).
  • The minimum is obtained by taking the smallest key in the map (min(freq)).

Maintaining this frequency map allows us to efficiently manage the sliding window and check its validity in (O(m)), where (m) is the number of distinct elements in the window.

Expanding and Validating the Window As we iterate through the array with the right pointer:

  • We expand the window by adding the current element to the frequency map.
  • This step increases the size of the window while keeping track of the current element frequencies.

After adding a new element, we check whether the window is valid by comparing the difference between the maximum and minimum elements:

  • If the difference exceeds 2, the window violates the condition.
  • To restore validity, we shrink the window by removing elements from the left until the condition is satisfied.

Complexity Analysis

  • Time Complexity:
    • Adding and removing elements from the frequency map take (O(m)), where (m) is the number of distinct elements in the window.
    • In the worst case, each element is added and removed from the map once, resulting in a time complexity of (O(n m)), where (n) is the size of the input array.
  • Space Complexity:
    • The frequency map stores counts of at most (m) distinct elements, requiring (O(m)) space.

Key Intuition The core intuition behind this approach is that overlapping ranges in the array can form a valid sequence where the maximum and minimum elements satisfy the condition. By maintaining the window and checking validity efficiently using a sorted frequency map, we can quickly determine the valid subarrays while avoiding redundant computations.

This makes the algorithm highly efficient compared to a brute-force approach, which would require checking every subarray individually.
