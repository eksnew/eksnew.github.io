Swin Transformer: Hierarchical Vision Transformer using Shifted Windows

Posted on 2024-12-05 Edited on 2024-03-21 In 论文阅读

ICCV 2021的Best Paper。一篇有年头的文章，但经典值得一读。

简单总结 借鉴了许多过往CNN网络里的设计理念，将CNN与Transformer进行了完美结合。

1. 引入

期望构建一个视觉中通用的backbone，证明Transformer在视觉领域的作用。

1.1. 难点

Transformer从NLP转移到CV领域具有挑战性：

  • 最小尺寸（感觉不是这个意思，有待阅读相关文献）。不同于单词，图像像素是很多的，这意味着可选择的最小尺寸很多。而现有Transformer-based架构中token的尺寸是固定的，这并不适合CV。
  • 图像分辨率。如语义分割的视觉任务中，高分辨率图像在Transformer中不好处理，因为自注意力的计算复杂度与图像尺寸呈二次关系。 当输入图片尺寸稍大，计算复杂度就变得难以接受了。

1.2. What to do

提出一种新backbone，能构建分层的特征图且计算复杂度与图像尺寸呈线性关系。

2. 模型与方法论

2.1. 结构细节

在Swin Transformer-Tiny中，原始图像输入。假设：

  • 分割。图像被分割为不重叠的patch，patch尺寸为，即每个patch的特征维度为；
  • 一个线性投影层。patch会被映射到预先设定好的维度（Swin-T中）。此时有个tokens。 在普通ViT中，3136的token长度绝对是不能接受的，但因为在Swin Transformer中一个窗口大小是，所以只有49个token。 Transformer的输入输出序列长度相同，故输出尺寸不变；
  • Patch Merging。产生分层表示，构建层级式Transformer。

关于图中未画出的分类头/检测头 不同于ViT，Swin Transformer中没有使用CLS token， 而是像CNN那样在最终得到后用Global Average Pooling将拉直为，最终得到。

2.2. 分层特征图

传统ViT将图片分成的patch尺寸是不变的（ViT中用的patch是的）， 每一层Transformer块看到的token都是固定倍数的下采样率()。 虽然ViT依然能够通过全局的自注意力操作达到全局的建模能力，但对多尺度特征把握较弱，而视觉下游任务中多尺度特征极为重要。

多尺寸特征 卷积神经网路的多尺寸特征主要来源于Pooling操作。 池化操作能增大每个卷积核能看到的感受野，从而使得池化后能抓住物体不同尺寸的特征。

类似地，Swin Transformer提出了类似Pooling操作的Patch Merging，将相邻的小尺寸patch合并为一个大patch，增加感受野。 在上图中，Swin Transformer从小尺寸patch开始，逐渐合并在更深Transformer层中的相邻patch来构建分层表示， 下采样率从刚开始的4倍变成8倍、16倍等。

像CNN里一样，通过这些分层特征图，接下来Swin Transformer可以更方便地用FPN、U-Net等网络进行密集预测。（大师，太妙了）

什么是密集预测？ ChatGPT：密集预测(Dense Prediction)指在计算机视觉中，对图像中的每个像素点进行预测或分类的任务。相比于传统的分类任务，密集预测任务不仅要输出图像的整体标签，还要在像素级别上对每个像素进行分类或预测，因此需要对图像中的每个位置进行处理。 密集预测常见的任务包括语义分割(Semantic Segmentation)、实例分割(Instance Segmentation)、图像分割(Image Segmentation)、光流估计(Optical Flow Estimation)等。这些任务要求模型能够对图像中的每个像素进行分类、分割或估计，以实现对图像更细粒度的理解和处理。

2.2.2. 为什么窗口自注意力机制能实现线性

Swin Transformer中，线性计算复杂度通过在分割图像的非重叠窗口（以红色框出）内计算自注意力实现。 因为每个窗口中patch数量是固定的，因此复杂度与图像大小成线性关系。

具体地说，为了减小序列长度，Swin Transformer在小窗口内计算自注意力，这样窗口大小固定，自注意力的计算复杂度便固定， 整张图片的计算复杂度则与图片大小呈线性关系。 假设每个窗口包含个patch（如前所述，一般为），则对于一张有个patch的图片， 标准多头自注意力和基于窗口的多头自注意力的计算复杂度为：

2.2.3. 窗口自注意力机制的缺点

窗口间没有通信，因此失去了全局自注意力。

2.3. Shifted Window方法

2.3.1. Swin Transformer block

Shifted Window是Swin Transformer的核心设计。Swin Transformer是通过将Transformer块中的标准多头自注意力(Multi-head Self Attention, MSA)模块替换为基于Shifted Window的模块而构建的，其他层保持不变，如下图所示。

其中，W-MSA和SW-MSA分别表示常规、移位窗口分割的多头自注意力模块， 这解决了标准Transformer架构中全局自注意力计算导致的复杂度与token二次相关的问题。

2.3.2. 窗口自注意力缺陷的解决——Shifted Window Partitioning Approach

如前述，线性计算复杂度通过在分割图像的非重叠窗口（以红色框出）内计算自注意力实现， 这意味着窗口自注意力机制缺乏跨窗口的连接，红色框内patch无法注意到其他红色框内的patch，这限制了其全局建模能力。 为了在引入跨窗口连接的同时保持非重叠窗口的高效计算，Swin Transformer提出一种移位窗口分区方法(shifted window partitioning approach)。

如上图所示，第一个模块采用常规窗口分割策略（从左上角像素开始分割），而第二个模块采用不同的配置： 将窗口从常规分割的窗口偏移像素。 窗口的多头自注意力+移动窗口的多头自注意力=Swin Transformer一个基本计算单元， 这也是模型中Swin Transformer块数量都是偶数的原因。 （灰色：最小patch，一般；红色：中型计算单元，一个窗口，默认有个最小patch。）

通过Shifted Window，一个patch不再局限于一个窗口内的交互，而是可以跨窗口进行交互， 因此Swin Transformer实现了两层相邻不重叠窗口间的连接，进而显著增强模型建模能力； 配合先前的Patch Merging，在Transformer的最后几层中，每个patch的感受野将变得很大， 此时窗口自注意力接近于一个全局的自注意力，最终达到一个即省内存效果又好的结果。

不同于其他Transformer-based架构中采用的滑动窗口法(Sliding Window)，Shifted Window方法的一个窗口内的所有query patch共享同一个key集，这有利于硬件中的内存访问。（为什么？）

ChatGPT: Shifted Window方法中，一个窗口内的所有 query patch 共享同一个 key 集，这是为了减少内存访问的重复次数和存储需求。在传统的滑动窗口法中，每个窗口的 query patch 都会与当前窗口范围内的所有 key 进行自注意力计算，这意味着同一个 key 会被多次访问。而在 Shifted Window 方法中，所有的 query patch 共享同一个 key 集，因此每个 key 只需要存储一次，并且在计算自注意力时可以重复使用，这减少了内存的占用和访问次数，提高了硬件的运行效率。

3. Swin Transformer的效率优化

3.1. 循环移位

Shifted Window虽好，但还是有一些问题。明显地，移动前后窗口的数量发生了改变，且窗口的大小也不相同， 那么就没法把这些窗口都压成一个batch去快速计算自注意力。 一个简单的解决方法是给这些不同大小的窗口进行padding，将它们填充为与最大窗口一样大的窗口，这样就可以压成一个batch了。 但这样会增加模型的计算复杂度，比如在前面的图中，原本只需要计算4个窗口，padding后就要计算9个窗口了。

如下图所示，Swin Transformer提出了循环移位的方法。在通过正常的Shifted Window方法得到9个窗口后， 先做一次循环移位，即移动图中的A、B、C三个部分以得到一个全新的大特征图，最后划分为4个窗口，这样窗口的数量就不会改变了。

但是这又产生了新的问题：循环移位后，图像的原始分布发生了改变，一些不应做自注意力的部分出现在了同一个窗口中（如天空现在可能在地面下方）， 这些部分不应该做自注意力。

3.2. 采用相对位置编码

在Transformer中，位置信息对于理解标记之间的空间关系至关重要。 传统的自注意机制依赖于绝对位置（即标记之间的固定距离），然而在视觉任务中，相对位置通常比绝对位置更相关， 理解图像中对象的相对排列比它们的绝对位置更重要。

Swin Transformer引入了相对位置偏差来解决这个问题，它不使用固定的位置编码，而是学习相对位置嵌入。 这些嵌入捕捉了标记之间的相对距离，使模型能够更有效地关注附近的标记。 相对位置偏差也降低了与绝对位置编码相比的计算成本。通过关注相对距离，模型可以有效地关注相关的标记，而无需考虑整个序列， 这种效率对于处理大图像并保持线性复杂度至关重要。

Swin Transformer的移位窗口方案实现了跨窗口连接，而相对位置偏差通过允许模型关注不同距离的相邻块，增强了这些连接， 这促进了更好的上下文建模和图像不同部分之间的信息交流。 总之，相对位置偏差提高了Swin Transformer在有效捕获空间关系方面的能力，特别是在相对位置比绝对坐标更重要的视觉任务中。

4.2. 消融实验

相对baseline而言，对分类任务，无论是shifted window还是相对位置编码影响均明显但不极度明显； 对下游任务（目标检测&语义分割），提升显著。

反思 密集型任务需要的特征对位置信息敏感，更需要周围上下文关系。 移动窗口提供的窗口间互相通信、在每个窗口都使用更准确的相对位置编码对此有帮助。
