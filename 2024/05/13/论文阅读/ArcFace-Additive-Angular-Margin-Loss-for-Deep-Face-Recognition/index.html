ArcFace: Additive Angular Margin Loss for Deep Face Recognition

Posted on 2024-05-13 Edited on 2024-07-06 In 论文阅读

这篇论文是18年CVPR。虽然这篇文章标题是ArcFace这篇论文，但我不是做这个大方向的，所以只部分记录其中内容。博客重点会放在几个损失函数的介绍和讨论上。

1. 文章大意

提出了一种用于人脸识别的新损失函数，叫ArcFace (Additive angular margin loss)。

1.1. ArcFace

ArcFace的思想(additive angular margin)与SphereFace、CosineFace有一定的共同点，论文中也对它们进行了比较。

1.2. Pipeline

常用的损失函数是Softmax loss based的，主要的pipeline为：

  1. 一个encoder用于学习数据的feature；

  2. 送进FC之类的网络和Softmax，最后用Cross-entropy计算loss。

2. 损失函数概述

2.1. Softmax与Softmax Loss

Softmax的公式很常见：
$$
\text{Softmax}(z_i) = \frac{e{z_i}}{\sum_{j=1}{N} e^{z_j}}
$$

其中，是模型输出的第个类别的原始分数， 是类别的总数。Softmax 函数能够将原始分数转换为0到1之间的概率值，且所有类别的概率之和为1。

softmax loss由softmax和交叉熵损失(cross-entropy loss)组合而成，全称是softmax with cross-entropy loss。其由三部分组成：Fully Connected Layer, Softmax Function 和 Cross-entropy Loss，pipeline如下图所示。

而Softmax loss的公式如下：

$$
\begin{aligned}
L_S &= -\frac{1}{N} \sum_{i=1}^{N} \log \left( \frac{e{z_{y_i}}}{\sum_{j=1}{N} e^{z_j}} \right) \
&= -\frac{1}{N} \sum_{i=1}^{N} \log \left( \frac{e{W_{y_i}T x_i + b_{y_i}}}{\sum_{j=1}^{N} e{W_{y_j}T x_i + b_{j}}} \right)
\end{aligned}
\tag{1}
$$

其中， 是样本的数量， 是模型对第 个类别的预测分数， 是第 个样本的真实标签。

问题
通常情况下，softmax loss仅仅能够使得特征可分，还不能够使得特征具有可判别性。

尽管有一些方法通过结合softmax loss和 contrastive loss，center loss去提高特征的可判别性，但是contrastive loss和center loss需要精心地构建图像对和三元组，不仅耗时而且构建的训练对会对识别性能影响很大。因此，我们需要一些改进。

3. ArcGeo loss

ArcGeo loss是在 ArcGeo: Localizing Limited Field-of-View Images using Cross-view Matching 中提出的。

然而，与人脸识别相反，在交叉视图匹配中，每个位置通常由单个空中-地面对表示。因此，在训练质心的位置期间，由于每个位置的样本数量非常有限，近似嵌入空间结构可能会与给定位置的实际模型输出不一致。这种错位会导致模型权重更新不正确，并可能解释使用 ArcFace 损失进行跨视图训练的低性能。
